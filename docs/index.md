# Documentation

Open Test Sandbox keeps the core generic and local-first. Start with the
shortest path for your role.

## New Users

- [Quick Start](quickstart.md): install dependencies, create a local Store, run
  the workbench, and verify the checkout.
- [API Case Format](api-case-format.md): write one runnable HTTP case and
  understand the Evidence files it creates.
- [Store Backends](store-backends.md): use SQLite locally and understand the
  optional team-store direction.

## Profile Authors

- [Profile Format](profile-format.md): manifest fields, split assets, audit,
  install, pack, import, and verify.
- [Profile Authoring Guide](profile-authoring.md): practical workflow for
  keeping team-owned test assets outside the core repository.

## Agent and CI Integrators

- [CLI and API Contracts](cli-api-contracts.md): discovery, single-interface
  reports, workflow reports, asynchronous batch reports, and failed-case
  Evidence lookup.
- [Release Checklist](release-checklist.md): local and CI gates before sharing
  a public tag.

## Project Overview

- [Core Capabilities, Skills, and Goals](core-capabilities-skills-goals.html):
